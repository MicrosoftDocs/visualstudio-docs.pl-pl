# <a name="color-value-reference-for-visual-studio"></a>Odwołanie do wartości koloru dla Visual Studio

Ten temat zawiera informacje dotyczące tworzenia nowych tokenów kolorów. Podczas tworzenia nowego interfejsu użytkownika możesz użyć podobnych kolorów z istniejącego interfejsu użytkownika, aby wybrać kolory dla każdego motywu głównego i nazwę koloru systemu dla tego elementu dla duży kontrast motywów.

Aby uzyskać informacje o implementacji dotyczące tworzenia nowych tokenów kolorów, zobacz [Narzędzia do tworzenia kolorów.](../../extensibility/internals/color-theming-tools.md)

Aby uzyskać tokeny kolorów do użycia z istniejącym interfejsem użytkownika, zobacz [Udostępnione kolory dla Visual Studio](../../extensibility/ux-guidelines/shared-colors-for-visual-studio.md).

Podczas korzystania z tego przewodnika należy pamiętać o:

- **Zawsze kopiuj jednocześnie kolory pierwszego planu i tła.** Nie wybieraj pierwszego planu z jednego elementu interfejsu użytkownika i tła z innego tylko dlatego, że podobają Ci się kolory.

- **Są to kody hex, a nie tokeny kolorów.** Nie można użyć rzeczywistych nazw tokenów kolorów dla tych elementów interfejsu użytkownika, ponieważ nie można kontrolować sposobu zmiany tokenów w przyszłości. Zamiast tego utwórz tokeny w ramach własnej kategorii i użyj wartości hue pokazanych w tabelach. Tokeny kolorów dla istniejącego interfejsu użytkownika można znaleźć w tece [Udostępnione kolory dla Visual Studio](../../extensibility/ux-guidelines/shared-colors-for-visual-studio.md).

- **duży kontrast są wyświetlane tylko dla porównania.** Użyj nazwy koloru systemu dla duży kontrast użytkownika i nie używaj na tych próbkach programu dooki ani s wyboru kolorów. duży kontrast w tym temacie odzwierciedlają duży kontrast #1 w systemie Windows. Kolory będą wyglądać inaczej w innych motywach duży kontrast motywach.

- **Aby uzyskać najlepsze wyniki, sparuj zestawy kolorów z interfejsem użytkownika podobnym do przykładu.**
