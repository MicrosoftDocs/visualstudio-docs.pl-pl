# <a name="ca2328-ensure-that-jsonserializersettings-are-secure"></a>CA2328: Upewnij się, że ustawienia JsonSerializerSettings zostały zabezpieczone

|Element|Wartość|
|-|-|
|CheckId|CA2328|
|Kategoria|Microsoft.Security|
|Zmiana podziału|Nieprzerwanie|

## <a name="cause"></a>Przyczyna

Ta zasada jest wyzwalana, gdy oba poniższe warunki mogą być spełnione dla wystąpienia [Newtonsoft.Json.JsonSerializerSettings](https://www.newtonsoft.com/json/help/html/T_Newtonsoft_Json_JsonSerializerSettings.htm) :
- Właściwość [TypeNameHandling](https://www.newtonsoft.com/json/help/html/P_Newtonsoft_Json_JsonSerializerSettings_TypeNameHandling.htm) jest wartością inną niż `None` .
- Właściwość [pomocą elementu SerializationBinder](https://www.newtonsoft.com/json/help/html/P_Newtonsoft_Json_JsonSerializerSettings_SerializationBinder.htm) ma wartość null.

Gdy wystąpienie [JsonSerializerSettings](https://www.newtonsoft.com/json/help/html/T_Newtonsoft_Json_JsonSerializerSettings.htm) jest używane w następujących okolicznościach:
- Zainicjowany jako pole klasy lub właściwość.
- Jako wartość zwracana metody.
- Używane jako ustawienia w metodzie [OnconvertNewtonsoft.Json.Js](https://www.newtonsoft.com/json/help/html/T_Newtonsoft_Json_JsonConvert.htm) .
- Używane jako ustawienia w metodzie [OnserializerNewtonsoft.Json.Js](https://www.newtonsoft.com/json/help/html/T_Newtonsoft_Json_JsonSerializer.htm) .

Ta reguła jest podobna do [CA2327](ca2327.md), ale analiza nie może określić, czy ustawienia są czasowo niezabezpieczone.

Domyślnie ta reguła analizuje całą bazę kodu, ale [można to skonfigurować](#configurability).

- [TypeNameHandling](https://www.newtonsoft.com/json/help/html/T_Newtonsoft_Json_TypeNameHandling.htm) `None` Jeśli to możliwe, użyj wartości TypeNameHandling.
- Przekształć dane serializowane. Po serializacji należy kryptograficznie podpisywać serializowane dane. Przed deserializacji Sprawdź poprawność podpisu kryptograficznego. Ochrona klucza kryptograficznego przed ujawnieniem i projektowaniem na potrzeby rotacji kluczy.
- Ogranicz typy deserializowane. Zaimplementuj [Newtonsoft.Jsniestandardowe na. Serializacja. ISerializationBinder](https://www.newtonsoft.com/json/help/html/T_Newtonsoft_Json_Serialization_ISerializationBinder.htm). Przed deserializacji z Json.NET upewnij się, że niestandardowe [ISerializationBinder](https://www.newtonsoft.com/json/help/html/T_Newtonsoft_Json_Serialization_ISerializationBinder.htm) są określone we właściwości [Newtonsoft.Json.JsonSerializerSettings. pomocą elementu SerializationBinder](https://www.newtonsoft.com/json/help/html/P_Newtonsoft_Json_JsonSerializerSettings_SerializationBinder.htm) . WNewtonsoft.Jszastąpione [. Metoda Serialization. ISerializationBinder. BindToType](https://www.newtonsoft.com/json/help/html/M_Newtonsoft_Json_Serialization_ISerializationBinder_BindToType.htm) , jeśli typ jest nieoczekiwany, zwraca `null` lub Zgłoś wyjątek w celu zatrzymania deserializacji.

## <a name="when-to-suppress-warnings"></a>Kiedy pominąć ostrzeżenia

Można bezpiecznie pominąć ostrzeżenie z tej reguły, jeśli:
- Wiadomo, że dane wejściowe są zaufane. Należy wziąć pod uwagę, że granica zaufania aplikacji i przepływy danych mogą ulec zmianie z upływem czasu.
- Wykonano jedną z tych środków ostrożności w zakresie [rozwiązywania naruszeń](#how-to-fix-violations).
- Wiadomo, że właściwość [Newtonsoft.Json.JsonSerializerSettings. pomocą elementu SerializationBinder](https://www.newtonsoft.com/json/help/html/P_Newtonsoft_Json_JsonSerializerSettings_SerializationBinder.htm) jest zawsze ustawiana, gdy właściwość [TypeNameHandling](https://www.newtonsoft.com/json/help/html/P_Newtonsoft_Json_JsonSerializerSettings_TypeNameHandling.htm) ma wartość inną niż `None` .

## <a name="configurability"></a>Określając

Jeśli uruchamiasz tę regułę z [analizatorów FxCop](install-fxcop-analyzers.md) (a nie ze starszą analizą), możesz skonfigurować analizę dla tej reguły.

### <a name="excluded-symbol-names"></a>Wykluczone nazwy symboli

Można skonfigurować, które części bazy kodu mają być wykluczone z analizy. Na przykład aby określić, że reguła nie powinna być uruchamiana na żadnym kodzie w typach o nazwie `MyType` , Dodaj następującą parę klucz-wartość do pliku editorconfig w projekcie:

```ini
dotnet_code_quality.CA2328.excluded_symbol_names = MyType
```

Dozwolone formaty nazw symboli w wartości opcji (oddzielone znakiem "|"):
  - Tylko nazwa symbolu (zawiera wszystkie symbole z nazwą, niezależnie od typu zawierającego lub przestrzeni nazw)
  - W pełni kwalifikowane nazwy w [formacie identyfikatora dokumentacji](https://github.com/dotnet/csharplang/blob/master/spec/documentation-comments.md#id-string-format)symbolu. Każda nazwa symbolu wymaga prefiksu rodzaju symboli, takiego jak "M:" prefix dla metod, "T:" prefiksu dla typów, "N:" prefiksu dla przestrzeni nazw itd.
  - `.ctor`dla konstruktorów i `.cctor` dla konstruktorów statycznych

Przykłady:

| Wartość opcji | Podsumowanie |
| --- | --- |
|`dotnet_code_quality.CA2328.excluded_symbol_names = MyType` | Dopasowuje wszystkie symbole o nazwie "MyType" w kompilacji
|`dotnet_code_quality.CA2328.excluded_symbol_names = MyType1|MyType2` | Dopasowuje wszystkie symbole o nazwie "MyType1" lub "MyType2" w kompilacji
|`dotnet_code_quality.CA2328.excluded_symbol_names = M:NS.MyType.MyMethod(ParamType)` | Dopasowuje określoną metodę "nomethod" przy użyciu podanego w pełni kwalifikowanego podpisu
|`dotnet_code_quality.CA2328.excluded_symbol_names = M:NS1.MyType1.MyMethod1(ParamType)|M:NS2.MyType2.MyMethod2(ParamType)` | Dopasowuje określone metody "MyMethod1" i "MyMethod2" z odpowiednimi w pełni kwalifikowanymi sygnaturami

Wszystkie te opcje można skonfigurować dla opcji tylko dla tej reguły, dla wszystkich reguł lub dla wszystkich reguł w tej kategorii (Security). Aby uzyskać więcej informacji, zobacz [Konfigurowanie analizatorów FxCop](configure-fxcop-analyzers.md).

## <a name="pseudo-code-examples"></a>Przykłady pseudo kodu

### <a name="violation"></a>Krocz

```csharp
using System;
using Newtonsoft.Json;
using Newtonsoft.Json.Serialization;

public class BookRecordSerializationBinder : ISerializationBinder
{
    // To maintain backwards compatibility with serialized data before using an ISerializationBinder.
    private static readonly DefaultSerializationBinder Binder = new DefaultSerializationBinder();

    public void BindToName(Type serializedType, out string assemblyName, out string typeName)
    {
        Binder.BindToName(serializedType, out assemblyName, out typeName);
    }

    public Type BindToType(string assemblyName, string typeName)
    {
        // If the type isn't expected, then stop deserialization.
        if (typeName != "BookRecord" && typeName != "AisleLocation" && typeName != "WarehouseLocation")
        {
            return null;
        }

        return Binder.BindToType(assemblyName, typeName);
    }
}

public class BookRecord
{
    public string Title { get; set; }
    public object Location { get; set; }
}

public abstract class Location
{
    public string StoreId { get; set; }
}

public class AisleLocation : Location
{
    public char Aisle { get; set; }
    public byte Shelf { get; set; }
}

public class WarehouseLocation : Location
{
    public string Bay { get; set; }
    public byte Shelf { get; set; }
}

public static class Binders
{
    public static ISerializationBinder BookRecord = new BookRecordSerializationBinder();
}

public class ExampleClass
{
    public BookRecord DeserializeBookRecord(string s)
    {
        JsonSerializerSettings settings = new JsonSerializerSettings();
        settings.TypeNameHandling = TypeNameHandling.Auto;
        settings.SerializationBinder = Binders.BookRecord;
        return JsonConvert.DeserializeObject<BookRecord>(s, settings);    // CA2328 -- settings might be null
    }
}
```
