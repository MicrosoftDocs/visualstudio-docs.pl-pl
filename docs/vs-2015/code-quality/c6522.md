# <a name="c6522"></a>C6522
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Ostrzeżenie C6522: Nieprawidłowa specyfikacja rozmiaru: wyrażenie musi być typu całkowitego  
  
> [!NOTE]
>  Ostrzeżenie to pojawia się tylko w przypadku kodu, który używa przestarzałej wersji kodu źródłowego języka adnotacji (SAL). Zaleca się, że port kodu, aby użyć najnowszej wersji SAL. Aby uzyskać więcej informacji, zobacz [przy użyciu adnotacji SAL w celu zmniejszenia defektów kodu C/C++](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md).  
  
 To ostrzeżenie wskazuje, że oczekiwany typ całkowity, ale niepoprawny typ danych został użyty. Można użyć właściwości adnotacji, które akceptują rozmiar parametru pod względem innego parametru, ale muszą używać poprawnym typie danych. Aby uzyskać listę właściwości adnotacji, zobacz [właściwości adnotacji](http://msdn.microsoft.com/en-us/f77b4370-6bda-4294-bd2a-e7d0df182a3d).  
  
## <a name="example"></a>Przykład  
 Poniższy kod generuje to ostrzeżenie:  
  
```  
// C  
#include <CodeAnalysis\SourceAnnotations.h>  
void f ([SA_Pre(ValidBytes="c")] char *pc, double c);  
  
// C++  
#include <CodeAnalysis\SourceAnnotations.h>  
using namespace vc_attributes;  
void f ([Pre(ValidBytes="c")] char *pc, double c);  
```  
  
 Aby poprawić to ostrzeżenie, użyj `size_t` dla `ValidBytesParam` typ danych parametru, jak pokazano w poniższym przykładowym kodzie:  
  
```  
// C  
#include <CodeAnalysis\SourceAnnotations.h>  
void f ([SA_Pre(ValidBytes="c")] char *pc, size_t c);  
  
// C++    
#include <CodeAnalysis\SourceAnnotations.h>  
using namespace vc_attributes;  
void f ([Pre(ValidBytes="c")] char *pc, size_t c);   
```
